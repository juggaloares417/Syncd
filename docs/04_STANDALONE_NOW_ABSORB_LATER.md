# Standalone Now, Absorb Later

## Rule
Syncd must launch as a standalone app now, but be architected so it can later be absorbed into Listening Party without a rebuild.

## Standalone Now
Syncd has its own:
- branding
- onboarding
- auth flow
- navigation
- profile layer
- games section
- wallet/subscription surfaces

## Absorb Later
Listening Party should later be able to absorb Syncd through:
- shared identity
- shared music graph
- shared messaging
- shared events engine
- shared wallet potential
- shared notifications

## Shared Platform Domains
- auth
- users
- profiles
- music identity
- messaging
- wallet
- notifications
- live events
- moderation
- settings

## Syncd-Specific Domains
- dating preferences
- vibe sync
- contact unlock
- Mic Match
- Karaoke Roulette
- romantic chemistry logic
